Copilot prompt (Option B: Week-based assessments with dynamic question count + edge cases)
You are implementing week-level assessments (one test per week) instead of per-item assessments.

Current system:

Backend: FastAPI + SQLAlchemy async
Roadmap endpoint returns week projections (weeks → skills/items)
Existing item assessment endpoints exist: /api/v1/assessments/items/{item_id} and submit/history
Learning plan persistence is one LearningPlanItem per skill; week segmentation is response-level projection
Goal:

For each week in a user’s roadmap, generate and persist a Week Assessment whose questions cover all concepts in that week, including edge cases.
Question count should be dynamic based on week scope.
Passing the week assessment (≥70%) unlocks progression and triggers next week assessment generation (staged generation).

2) Question schema (week-based)

{
  "question": "string",
  "options": ["A","B","C","D"],
  "correct_index": 0,
  "explanation": "string",
  "tags": ["subtopic:Prompt Engineering", "skill:Generative AI", "type:edge_case"]
}
3) Dynamic question count formula
Compute per week:

Gather week scope:

list of items in that week (skill names)
all subtopics/sub-subtopics in that week (from DB for each LearningPlanItem)
week_hours estimate (sum of item estimated hours or use week projection hours)
Compute:

minQ = 8, maxQ = 20

base = 8

extra = ceil(total_subtopics / 3) + ceil(week_hours / 5)

Q = clamp(minQ, maxQ, base + extra)

Coverage requirements in prompt:

At least 1 question per skill if possible
At least 1 question per major subtopic cluster
At least 20% “edge case/failure-mode” questions (round up, min 2)
At least 30% scenario-based applied questions
